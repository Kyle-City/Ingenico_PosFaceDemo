# 已知问题

## 百度官方 Demo / DX8000 PoC

### 未解决问题

1. 人证核验模式方向仍逆时针旋转 90 度。
2. 属性模式方向仍逆时针旋转 90 度。
3. DX8000 摄像头为固定焦点，最高可用预览分辨率为 640x480；在老旧 POS 上容易出现 blur 质量项偏高。
4. 当前 PoC 将运行配置中的 blur 阈值临时调整为 `0.995`，仅用于老旧 POS 闭环验证，不可作为生产参数。
5. `DX8000_*` 诊断日志仍保留在官方 Demo 中，便于继续定位授权、相机、质量管线和注册页显示问题；生产集成前需重新评估日志级别与内容。

### 当前不处理范围

1. 不处理人证核验模式方向问题。
2. 不处理属性模式方向问题。
3. 不连接真实支付。
4. 不处理真实客户数据。
5. 不提交或记录 Secret Key、KeyStore、人脸图片、设备指纹或其他敏感标识。

### 历史问题

1. 摄像头权限弹窗问题。

## PosFaceDemo / DX8000 离线人脸识别核心 PoC

### 已知限制

1. `qualityMinFaceWidth=200` 是基于 DX8000 固定焦点摄像头、640x480 输入和有限实机样本得到的 PoC 标定值，不是生产阈值。
2. RGB 活体分数和本地 1:N 相似度分数只来自有限测试，不构成正式安全评估。
3. DX8000 摄像头为固定焦点，当前最高可用预览分辨率为 640x480，距离、光线和姿态会显著影响质量门控。
4. `FaceSearch.pushPersonById(localId, feature)` 对传入 feature ByteArray 的所有权语义仍未确认；当前为避免破坏内存库，SQLite 读取后 push 的数组暂不主动清零。
5. Debug 授权入口、模型初始化入口和诊断入口仅用于 PoC 与调试，不得进入正式发布版本。

### 正式化前检查项

1. 重新评估质量阈值、RGB 活体阈值和 1:N 识别阈值。
2. 明确本地特征数据的生命周期、加密、删除、备份和合规策略。
3. 移除或隔离 Debug-only 授权、激活和诊断 UI。
4. 建立更完整的活体攻击、误识别、拒识、弱光和多人场景测试。
